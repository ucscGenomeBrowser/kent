Description

This track shows regions where transcription factors, proteins responsible for modulating gene transcription, bind to DNA as assayed by ChIP-seq (chromatin immunoprecipitation with antibodies specific to the transcription factor followed by sequencing of the precipitated DNA). The peaks of transcription factor binding underyling this track were computed using a uniform pipeline that uses the variation between the two replicates to guide peak thresholds. This track combines data from 91 cell types and 189 transcription-factor targeting antibodies into a summary display. Additional views of this data and additional documentation on the methods used to generate the data underlying this track are available from the ENCODE Uniform TFBS track description.

Display Conventions

A gray box encompasses the peak clusters of transcription factor occupancy. The darkness of the box is proportional to the maximum signal strength observed in any cell line contributing to the cluster. The name to the left of the box is the transcription factor. The letters to the right represent the cell lines where a signal is detected. The darkness of the letter is proportional to the signal strength in the cell line. Abbreviations starting with capital letters designate ENCODE cell types identified for intensive study - Tier 1 and Tier 2 - while those starting with lowercase letters designate Tier 3 cell lines. Click on an item in the track to see the cell lines spelled out.

Methods

Peaks of transcription factor occupancy from uniform processing of ENCODE ChIP-seq data at a .9% FDR threshold from datasets provided by the ENCODE Analysis Working Group that passed the integrated quality metric (see "Quality Control" section of Uniform TFBS) were clustered using UCSC hgBedsToBedExps. The cluster score is the highest score for any peak contributing to the cluster. The input signal values were multiplied by a normalization factor calculated as the ratio of the maximum score value (1000) to the signal value at one standard deviation from the mean, with values exceeding 1000 capped at 1000. This has the effect of distributing scores up to mean plus one 1 standard deviation across the score range, but assigning all above to the maximum score.

For additional documentation on the methods used to generate the data underlying this track please review the ENCODE Uniform TFBS track description.

Release Notes

Release 3 (August 2013) of this track includes more datasets (690 total, vs. 486 in the previous release) representing all ENCODE transcription factor ChIP-seq passing quality assessment through the ENCODE March 2012 data freeze. The peaks used to generate these clusters were called with less stringent thresholds than used during the January 2011 uniform processing shown in Release 2 of this track. The peaks that were used to generate the clusters in this version are displayed as individual tracks in the ENCODE Uniform TFBS track, which is available along with the primary data tracks at the ENC TF Binding Supertrack page. The clustering for V3 is based on the transcription factor target, and so differs from V2 where clustering was based on antibody.

For this release, a new track table format, 'factorSource' was used to represent the primary clusters table and downloads file, wgEncodeTfbsClusteredV3. This format consists of standard BED5 fields (see File Formats) followed by two fields containing comma-separated lists. The first list field contains numeric identifiers for experiments, keyed to the wgEncodeTfbsClusteredInputsV3 table, which includes such information as the experiment's underlying Uniform TFBS table name, factor targeted, antibody used, cell type, treatment (if any), and laboratory source. The second list field contains the scores for the corresponding experiments. In the downloads directory there is a BED file wgEncodeRegTfbsClusteredWithCellsV3 that provides the cluster score followed by a comma-separated list of cell types.

Data Release Policy

While primary ENCODE data was subject to a restriction period as described in the ENCODE data release policy, this restriction does not apply to the integrative analysis results, and all primary data underyling this track have passed the restriction date. The data in this track are freely available.
